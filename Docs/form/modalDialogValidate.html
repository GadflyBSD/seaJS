Dialog弹框表单验证

关于

表单验证是Web开发中最常见的流程，一般分为前端js验证，和后端服务器脚本验证两部分。前端js验证不是必须，但为了加强用户体验，却又往往是必不可少。但很多重复的表单验证却花去了开发人员大量宝贵时间。jQuery Validation Plugin为一款简单好用的表单验证插件.。
兼容IE 6, IE 7, IE 8, FF 3.6, Chrome 6.0, Safari 5.0, Opera 10.62。

通过HTML文档data属性

仅仅通过向页面元素添加data-sea="Dialog" 就可以为表单赋予Validation表单验证功能。
< form action="URL" method="ajax/debug/post/get" data-sea="Validate" data-msgTime="2" data-async="true" data-callback="callback" >
以下是其他相关属性：

  • data-dialog 属性接受一个字符串，指定弹框类型，提供三种弹框类型：showModal（模态弹出框）、dialog（普通弹出框）和FloatLayer（气泡浮层），默认是“showModal”；
  • data-float 属性接受一个字符串，气泡浮层浮动于所指定元素的ID，仅当【data-dialog="FloatLayer"】时需要设置；
  • data-target 属性接受一个字符串，指定弹出表单的模板ID。默认为“modalDialog”；
  • data-title 属性接受一个字符串，指定弹出表单的标题栏内容，默认值是“My Modal Dialog Title”；
  • data-callback 属性接受一个回调函数的名称，默认是空，如指定了回调函数，需要定义该回调函数；在弹框表单中这里的回调函数在表单验证提交成功触发关闭弹框前触发。作为弹框表单的回调函数时本回调函数提供三个参数：data（表单提交数据序列化后的字符串）, note（Ajax返回的Json对象）, mess（提示框对象，用于操作提示框）

特别注意：在哪里和如何自定义回调函数?

请在seaJS的入口位置，使用jQuery.extend(object)的方式设定回调函数，例如： 

var SeaOptions = ({
	Dialog:{
		tmpl:true,
		width:'850px',
		tmplData:{
			ordersn:'这里是jQuery TMPL 传递变量'
		}
	}
});

小技巧：表单提交后如何阻止关闭?

答：使callback回调函数return true即可!
