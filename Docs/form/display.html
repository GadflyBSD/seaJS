表单控件展示

关于

在一些需要展示程序源码的地方，有时候需要代码语法高亮，以便方便浏览。
jQuery插件seaSnippet是一个基于SHJS的用于程序源码语法高亮显示的插件。它提供了一种快速和简便的方式在HTML文档上对各种源码片段进行高亮显示。
插件兼容IE 6, IE 7, IE 8, FF 3.6, Chrome 6.0, Safari 5.0, Opera 10.62。
插件自带有39种样式可以使用。

通过HTML文档data属性

仅仅通过向页面元素添加data-sea="seaSnippet" 就可以为其赋予代码语法高亮效果，以便方便浏览。
以下是可选属性：

  • data-target 属性接受一个选择器，将选择器内的代码进行语法高亮输出；
  • data-language 属性接受一个高亮显示的语言名称，高亮显示按该语言语法进行显示，默认是HTML；
  • data-style 属性接受一个Query插件seaSnippet代码语法高亮效果的样式文件名，默认是random（随机）；
  • data-clipboard 属性接受一个剪切Flash控件的地址，默认是../SeaJS/seaSnippet/ZeroClipboard.swf，设定为false则不显示剪切；
  • data-showNum 属性指定是否显示行号。默认是true；
  • data-collapse 属性指定是否折叠代码，默认是false；当选择true时，data-showMsg、data-hideMsg属性有效；
  • data-showMsg 属性指定折叠代码时，展开折叠显示的按钮显示的字样。当data-collapse为true时有效，默认是“展开示例代码”；
  • data-hideMsg 属性指定折叠代码时，关闭折叠显示的按钮显示的字样。当data-collapse为true时有效，默认是“隐藏示例代码”。

  • 日历组件

仅仅通过向页面表单元素添加data-sea="Calendar" 就可以为其赋予日历组件。
以下是可选属性：

  • changeMonth 属性接受一个选择器，将选择器内的代码进行语法高亮输出，默认是true；
  • changeYear 属性接受一个高亮显示的语言名称，高亮显示按该语言语法进行显示，默认是true；
  • dayNamesMin 属性接受一个Query插件seaSnippet代码语法高亮效果的样式文件名，默认是“["日","一","二","三","四","五","六"]”；
  • monthNamesShort 属性接受一个剪切Flash控件的地址，默认是“["一月","二月","三月","四月","五月","六月","七月","八月","九月","十月","十一月","十二月"]”；
  • yearSuffix 属性指定是否显示行号。默认是“ 年 ”；
  • showMonthAfterYear 属性指定是否折叠代码，默认是“!0”；
  • yearRange 属性指定折叠代码时，展开折叠显示的按钮显示的字样。当data-collapse为true时有效，默认是“c-3:c+0”；
  • dateFormat 属性指定折叠代码时，关闭折叠显示的按钮显示的字样。当data-collapse为true时有效，默认是“yy年mm月dd日”。
<input type="text" name="Datepicker" placeholder="请输出年月日" data-toggle="seaCalendar">

  • 地区三级联动下拉选单

仅仅通过向页面表单元素添加data-sea="Reside" 就可以为其赋予地区三级联动下拉选单。
以下是可选属性：

  • data-default 属性接受一个JSON格式数组，用作选单默认值，默认是“[0,0,0]”；
  • data-reside 属性接受一个JSON格式数组，用作选单默认ID，默认是“['resideprovince','residecity','residedist']”；
  • data-class 属性接受一个CLASS选择器，用作选单添加class，默认是“form-control”；

	<div class="col-sm-4 city resideprovince"></div>
	<div class="col-sm-4 city residecity"></div>
	<div class="col-sm-4 city residedist"></div>
</div>

  • Kindeditor富文本输入框

仅仅通过向页面表单元素添加data-sea="Kindeditor" 就可以为其赋予Kindeditor富文本输入框组件。
以下是可选属性：

  • width 属性接受一个字符类型数据，编辑器的宽度，可以设置px或%，比textarea输入框样式表宽度优先度高。默认是“670px”；
  • height 属性接受一个字符类型数据，编辑器的高度，只能设置px，比textarea输入框样式表高度优先度高。默认是“300px”；
  • filterMode 属性接受一个布尔型数据，true时根据 htmlTags 过滤HTML代码，false时允许输入任何代码。默认为True；
  • allowFileManager 属性接受一个布尔型数据，true时显示浏览远程服务器按钮。默认为True；
<textarea data-sea="Kindeditor" name="center"></textarea>

  • Number Picker 数字按钮组

仅仅通过向页面表单元素添加data-sea="NumberPicker" 就可以为其赋予Kindeditor富文本输入框组件。
以下是可选属性：

  • min 属性接受一个布尔型或数字类型数据，为False时不限最小值，当为0时，最小值为0。默认是“false”；
  • max 属性接受一个布尔型或数字类型数据，为False时不限最大值，当为0时，最大值为0。默认是“false”；
  • value 属性接受一个数字类型数据，输入按钮组呈现时的默认值。默认为0；
  • step 属性接受一个数字类型数据，输入按钮组改变时的步进。默认为1；
  • name 属性接受一个字符类型数据，代表输入框的name属性值。默认是“NumberPicker”；
  • format 属性接受一个布尔型或字符类型数据，代表数据是否格式化后呈现，可以是‘#.##’。默认是“false”；
  • editable 属性接受一个布尔型，代表输入框是否可以手工编辑。默认是“true”；
  • negative 属性接受一个布尔型数据，是否允许负值。默认为True；
  • addText 属性接受一个字符类型数据，增加按钮的字符显示。默认为“+”；
  • subText 属性接受一个字符类型数据，减少按钮的字符显示。默认为“-”；
  • formatter 属性接受一个函数或函数名，用于输出值带单位的情况，函数接收一个val参数，代表当前值； 
    formatter: function(val){
	return "$ "+val;
}
  • beforeIncrease 属性接受一个函数或函数名，用于点击增加按钮后，在实际增加前的回调函数；
  • afterIncrease 属性接受一个函数或函数名，用于点击增加按钮后，在实际增加后的回调函数；
  • beforeDecrease 属性接受一个函数或函数名，用于点击减少按钮后，在实际减少前的回调函数；
  • afterDecrease 属性接受一个函数或函数名，用于点击减少按钮后，在实际减少后的回调函数；；
  • beforeChange 属性接受一个函数或函数名，用于数值改变前的回调函数；
  • afterChange 属性接受一个函数或函数名，用于数值改变后的回调函数；
  • onMin 属性接受一个函数或函数名，用于当数值达到最小值时的回调函数；
  • onMax 属性接受一个函数或函数名，用于当数值达到最大值时的回调函数；
