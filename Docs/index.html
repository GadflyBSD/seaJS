SeaJS Javascript组件库

前言

JavsScript模块化已成为发展趋势，由阿里、腾讯等公司共同维护的Sea.js，无疑是前端JS模块化方面最有力的支持者，其发展势头强劲，越来越多的网站正在受益其中。将常用的JS功能整理成库，于是就有了SeaJS组件库。

简介

让组件呼之即来，再也不需要写 script 标签来引用某个功能相关的js文件了。如有相关的css文件， Sea.js 也会自动帮你引入。只要是你看好的JS功能，基本上都可以成为这里的组件。

每个组件都是Sea.js的模块，所以组件的使用依赖于Sea.js，也就是说要使用这些组件，需要在页面上先引入Sea.js。

<div data-sea="Plugin_1" data-config="Plugin_1_Option" data-option="P1_order1:'P1_value1',P1_order2:'P1_value2'"></div>
<div data-sea="Plugin_2" data-config="Plugin_2_Option" data-option="{P2_order1:'P2_value1',P2_order2:'P2_value2'}"></div>
<div data-sea="Plugin_3" data-config="Plugin_3_Option" data-option="({P3_order1:'P3_value1',P3_order2:'P3_value2'})"></div>
<script type="text/javascript">
var SeaOptions = ({
	Plugin_1_Option: {
		param_1: 'value_1',
		param_2: 'value_2',
		...
		param_n: 'value_n',
		callback: function(){
			some code ...
		}
	},
	Plugin_2_Option: {
		param_1: 'value_1',
		param_2: 'value_2',
		...
		param_n: 'value_n',
		callback: function(){
			some code ...
		}
	},
	Plugin_3_Option: {
		param_1: 'value_1',
		param_2: 'value_2',
		...
		param_n: 'value_n',
		callback: function(){
			some code ...
		}
	}
});
seajs.use(['seajs-control', 'bootstrap', '其他需加载模块...'], function(){
	$(document).seaLoad();
});
</script>
<div data-sea="Plugin_1" data-config="Plugin_1_Option" data-option="Plugin_1_Orther"></div>
<div data-sea="Plugin_2" data-config="Plugin_2_Option" data-option="Plugin_2_Order"></div>

其中 config/1.1.1/config 是全局的配置文件，在这个文件当中对Sea.js进行了扩展。Sea.js本身没有自动加载组件的功能，为了简化组件的使用，我们给Sea.js扩展了这个功能。

组件开发使用了 Grunt http://gruntjs.com/。Grunt 是针对Javascript项目的一个自动化工具，包括初始化文件、代码审查、单元测试，合并、压缩等一系列常用功能，还有其他插件，也可以自定义插件。总之，这是一个很牛X的工具。

超级使用方法

给标签增加一个属性(data-sea="组件名")即可，比如需要调用日历组件（在引入jquery和Sea.js之后），仅需要给input标签增加一个 data-sea="calendar" 的属性，日历就可以工作了。如下

<input type="text" data-sea="calendar"/>

普通使用方法

除了上面这种超级简单的使用方式，一般情况下会这么使用，比如使用 underscore 组件

seajs.use(['underscore'], function(_){
    _.each([1, 2, 3], alert);
})

注意，组件都是有版本的，如 seajs.use(['underscore/1.4.4/underscore'],...) 或 seajs.use(['underscore@1.4.4'],...) ，如果不指定版本seajs.use(['underscore'],...)，则默认使用最新版本。

组件开发注意事项

  • 尽量避免使用jQuery的live方法(jQuery1.9版本不再支持这个方法)，以及on\off(需1.7+)
  • 尽量避免使用$.browser，jQuery1.9版本不再支持此对象
  • 组件需要标明依赖的jQuery版本
  • 组件需要有友好的演示示例
  • 组件组件名称必须符合js变量命名（比如不能包含“-”等）
  • 如果有css文件，css文件和js文件命名要相同（除了后缀.css的部分）。css文件也放在src目录，如有很多文件，在scr目录创建子目录即可。可参考autocomplete和artdialog两个组件的src目录。
