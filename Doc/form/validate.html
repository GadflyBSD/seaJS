jQuery 插件Validation 表单验证

关于

表单验证是Web开发中最常见的流程，一般分为前端js验证，和后端服务器脚本验证两部分。前端js验证不是必须，但为了加强用户体验，却又往往是必不可少。但很多重复的表单验证却花去了开发人员大量宝贵时间。jQuery Validation Plugin为一款简单好用的表单验证插件.。
兼容IE 6, IE 7, IE 8, FF 3.6, Chrome 6.0, Safari 5.0, Opera 10.62。

通过HTML文档data属性

仅仅通过向页面元素添加data-sea="Validator" 就可以为表单赋予Validation表单验证功能。
< form action="URL" method="ajax/debug/post/get" data-sea="Validate" data-msgTime="2" data-async="true" data-callback="callback" >
以下是其他相关属性：

  • action 属性接受一个URL地址，指定表单提交方向；
  • method 属性接受一个字符串，在原有method属性基础上扩展增加了ajax（Ajax方式提交）、debug（仅验证不提交），默认是ajax；
  • data-async 属性接受一个布尔值，指定表单验证是否使用异步加载方式，当为True时，自动为提交按钮添加disabled属性，任何一个input,select,textarea元件获得焦点时触发加载，并将提交按钮的disabled属性删除。默认为True；
  • data-msgTime 属性接受一个数字值，代表消息提示框多长时间后消失，默认值是5秒；
  • data-callback 属性接受一个回调函数的名称，默认是空；本回调函数提供三个参数：data（表单提交数据序列化后的字符串）, note（Ajax返回的Json对象）, mess（提示框对象，用于操作提示框）

注意：在弹框表单中此处定义的回调函数在表单验证提交成功触发关闭弹框后触发。如需要在弹框关闭事件前触发，请在弹框处指定回调函数。

特别注意：在哪里和如何自定义回调函数?

请在seaJS的入口位置，使用 jQuery.extend(object)的方式，或者 function callback(){}方式设定回调函数，例如： 

seajs.use(['seajs-control', 'bootstrap'], function(){
	function callback0(){
		some code ...
	}
	$.extend({
		callback: function(){
			some code...
			alert(101)
		}
	})
})
